# Genus Technologies Interview Preparation

## Interview Style (from Joanie)

**Jimar and Alyssa are precise and focused.**

### Communication Tips

- **Stay on topic** — Don't veer into unrelated areas
- **Periodically check in** — "Did I answer your question?" 
  - Mirrors how they want someone to communicate with clients
- **Listen to the full question** before answering
- **Demonstrate passion for tech** and continuous learning

---

## What They're Looking For

### Technical Evaluation

1. **End customer / end user experience**
   - How do you think about the user?
   - What problems are you solving?

2. **Supporting applications in Production**
   - Real-world deployment experience
   - Monitoring, logging, debugging in production
   - Handling production issues

3. **Component design (subsystems / APIs / libraries)**
   - How do you architect systems?
   - Design trade-offs
   - Scalability considerations

4. **Design patterns in practice**
   - Not just knowing theory
   - How you apply patterns to real problems
   - Trade-offs and alternatives

5. **Troubleshooting and fixing performance issues**
   - Profiling and analysis
   - Root cause identification
   - Systematic debugging approach

6. **Database table / query / stored procedure design**
   - Schema design
   - Query optimization
   - Performance considerations

### Soft Skills Evaluation

1. **Client-facing experience**
   - How you interact with non-technical stakeholders
   - Communication clarity

2. **Strong communication**
   - Written and verbal
   - Explaining technical concepts clearly

3. **Critical thinking**
   - Structured problem decomposition
   - Clear reasoning
   - Demonstrating thought process

---

## Interview Preparation Strategy

### Technical Preparation

1. **Review [Design Patterns](../../02-concepts/design-patterns.md)**
   - Be ready with real examples from your work
   - Discuss trade-offs

2. **Study [SQL and Database](../../03-languages-frameworks/sql-database.md)**
   - Query optimization
   - Schema design decisions

3. **Understand [SOLID Principles](../../02-concepts/solid.md)**
   - How they guide your design decisions

4. **Prepare production examples**
   - Problems you've encountered
   - How you debugged and fixed them
   - Performance optimizations you've made

### Communication Preparation

1. **Practice explaining technical concepts simply**
2. **Have concrete examples ready** from your work
3. **Think about user experience** in your solutions
4. **Be ready to discuss** how your code/design impacts the end user

---

## Interview Topics You Might Encounter

### System Design
- How would you architect a system to handle X?
- What trade-offs did you make?
- How would you handle performance at scale?

### Production Support
- Tell me about a production issue you debugged
- How did you identify the root cause?
- What did you learn?

### Performance Optimization
- How do you approach a slow system?
- What metrics do you look at?
- How did you measure improvement?

### Design Decisions
- Why did you choose this architecture?
- What alternatives did you consider?
- What would you do differently now?

---

## Key Points to Emphasize

✅ **End-user focus** — Always tie technical decisions to user value  
✅ **Production experience** — Show you understand real-world constraints  
✅ **Thoughtful design** — Demonstrate structured thinking  
✅ **Continuous learning** — Personal projects, staying current  
✅ **Clear communication** — Explain technical concepts accessibly
