# Personal Software Development Knowledge System

A comprehensive, organized guide to software development concepts, tools, frameworks, and best practices. Structured into three semantic blocks: **Learning**, **Professional**, and **Personal**.

---

## 📚 Structure

The knowledge system is organized into three semantic categories:

### [01-Learning](./01-learning/) — Courses, Concepts, and Frameworks

Educational resources, foundational concepts, and deep dives into languages and frameworks.

- **[01-Courses](./01-learning/01-courses/)** — Course notes from Frontend Masters and other programs
  - Enterprise UI Development (Steve Kinney)
  - Complete Intro to React, v9 (Brian Holt)
  - Complete Intro to SQL & PostgreSQL
  - AI for Software Engineers
  
- **[02-Concepts](./01-learning/02-concepts/)** — Foundational programming principles
  - General Principles, OOP, SOLID, Design Patterns, Testing
  
- **[03-FEM-Courses](./01-learning/03-fem-courses/)** — Structured Frontend Masters courses
  - C# and .NET: From Beginner to Professional (6 modules)
  
- **[04-Languages-Frameworks](./01-learning/04-languages-frameworks/)** — Language and framework deep dives
  - SQL and Database Development, ASP.NET MVC, CSS

### [02-Professional](./02-professional/) — Interviews, Tools, and Reference

Professional development, interview preparation, development tools, and reference materials.

### [03-Personal](./03-personal/) — Activities, Projects, and Growth

Personal development, project tracking, and activity logging.

## 🎯 How to Use This Repository

1. **For Learning** — Start with [01-Learning](./01-learning/02-concepts/) for foundational concepts, then dive into specific courses or frameworks

2. **For Reference** — Use the table of contents above to jump to what you need. All files are cross-linked

3. **For Interviews** — Review [02-Professional/01-Interview-Prep](./02-professional/01-interview-prep/) before technical interviews

4. **For Tools** — Check [02-Professional/02-Tools](./02-professional/02-tools/) when setting up new projects or workflows

5. **For Progress** — See [TODO.md](./TODO.md) for incomplete sections and courses to expand

6. **For Activity Log** — Your daily summaries are automatically tracked in [03-Personal/01-Activity-Log](./03-personal/01-activity-log/)

## 🔗 Cross-File Navigation

All files use relative markdown links for easy navigation in VS Code and GitHub:
- `[Link text](./file.md)` — Links to a file in the same folder
- `[Link text](../other-section/file.md)` — Links across folders
- `[Link text](./file.md#section-heading)` — Links to specific sections

---

## 🤖 Claude Code Integration

At the end of most Claude Code sessions, you'll be prompted:

> "What did you accomplish today? (Add to activity log)"

Your response is automatically summarized and appended to the current activity log. This keeps your personal progress record up-to-date without extra effort.

See [CLAUDE.md](./CLAUDE.md) for the automation details.
